# 01_strategy｜戦略骨子

## Zeims 戦略メモ（骨子）

GM: 川合（合同会社雲孫として実行）

- ICP（1行）: 従業員10〜50名の税理士/会計事務所で、申告繁忙期の業務圧縮を最優先する代表/担当者。
- 約束する価値（1文）: 申告関連の定型作業を自動化し、初月から月20〜30時間の工数を削減する。
- 価格・前受け: 月額 ◯◯万円（仮）、初回前受け50〜100％必須。割引は割引ルール表の範囲のみ。
- TTFV目標: 7日以内に最初の自動化ジョブを稼働させ、削減時間を可視化。
- 主要KPI案: 有料β社数、ARPA、前受率、TTFV中央値、1社あたりCS工数（h/月）。

# 02_offer｜価格・オファー

## 価格モデル（案）
- ベース月額 + ユーザー課金（相場: 月5,000〜10,000円 + ユーザー追加300円〜/月）
- ターゲット優先は Mid/Large（小規模は限定プランで対応）
  - Mid (5-29名): 月4〜7万円 + ユーザー×300円（10ユーザー込みを検討）
  - Large (30-200名): 月12〜20万円 + ユーザー×300円（最低席数設定を検討）
  - Small (1-4名): 月1.5〜2.5万円 + ユーザー×500円（軽量プラン、カスタマイズなし）
- エンタープライズ/高度案件
  - 初期オンボーディング費用: 20〜50万円（根拠ログ/セキュリティ設計/研修込み）
  - オプション: 監査ログテンプレ導入、国際税務データ連携 PoC

## 前受け
- 原則 50〜100% 前受け（PoC/β は前受け100%、エンタープライズは初期費で回収）。

## 割引ルール（案）
- コミュニティ/Dojo経由（BAAO経由）の紹介は初月10-20%ディスカウント上限。
- 年契約一括支払いで月額2ヶ月分ディスカウントを検討。

## 価値メッセージ（営業用）
- 調査リードタイム40%短縮 / レビュー工数30%削減（要検証）。
- 根拠リンク付き回答・事務所ルール適用・監査ログ対応が差別化軸。
- freee/MFの汎用エージェントではカバーしにくい「根拠提示×深度」を強調。

# brainbase｜MVV（共通リポジトリのOS）

## Mission（創業の理由｜今〜3年）
創業者が現場から抜けても、全事業が同じOSで動き、週次30分のレビューだけで運営が回る状態をつくる。判断基準・ナレッジ・タスク・ブランド資産を brainbase に一元集約し、誰が見ても同じ判断ができるようにする。

## Vision（未来像｜3〜5年）
brainbase 上に、すべての事業（UNSON / BAAO / SalesTailor / TechKnight / Zeims / Aitle）が、共通の構造・KPI・役割分担（RACI）・ブランド資産で接続される。  
その結果として、**「人が抜けても回る」「新規事業が90日で立ち上がる」**状態を標準化する。

## Values（現在の行動指針）
- 一元管理：情報は brainbase に集約し、リンクで参照する。二重管理はしない。
- 30分レビュー基準：週次レビューで30分あれば追える粒度で書く。迷ったら削る。
- テンプレ優先：テンプレ化・再利用可能な形を最優先する。例外対応は「追加コスト」として扱う。
- 数字で回す：KPIとゲート（次のステージに進む条件）を定義し、感覚ではなくデータで判断する。
- 前受けとルール徹底：前受け・SLA・割引禁止ルールを明文化し、例外は必ず決裁ログを残す。
