## 01_strategy｜戦略骨子

### プロダクト概要（1文）
毎週のAIアップデートを**実務の言葉**で解釈し、次の一手まで落とし込む会員制コミュニティ。

### ICP（1行）
AI活用に関心が高い事業責任者・リードエンジニア層で、自社へのAI導入を検討している中小〜ミッドマーケット企業の意思決定者。

### 約束する価値（1文）
最新の潮流を「社内で転送できる形」で掴み、PRIME・CONSULTで合意形成、BOOST・RECIPESで実装まで一気通貫で支援。

### プログラム体系

#### 継続課金プログラム

| プログラム | 目的 | 主な提供 | 月額（税抜） | 年額（税抜） |
|-----------|------|---------|------------|------------|
| **稽古場 BASIC** | 最新と実務解釈を毎週アップデート | 30分解説＋30分Q&A、ノート、アーカイブ | **2,980円** | **29,800円** |
| **稽古場 ACTIVE** | BASIC＋過去アーカイブ全文検索 | BASIC特典＋全アーカイブ＋Flow別フィード | **9,800円** | **98,000円** |
| **師範会 PRIME** | 先行事例・見学で意思決定を加速 | クローズド会、現場見学、ピアレビュー、**全サービス20%割引** | **196,000円** | **200万円** |

#### 都度課金サービス（PRIME会員は20%割引）

| サービス | 目的 | 料金（税抜） | PRIME価格 |
|---------|------|------------|----------|
| **CONSULT クイック** | 単発相談（60分×1回） | **15万円** | **無料** |
| **CONSULT スタンダード** | 稟議支援（90分×2回＋稟議素材） | **35万円** | **28万円** |
| **型録 RECIPES** | そのまま配れる業務手順書 | **100–150万円/本** | **80–120万円** |
| **立合い BOOST** | 4–8週で可視ROI | **400–800万円** | **320–640万円** |

> **CONSULT→BOOST移行特典**: CONSULT実施後3ヶ月以内にBOOSTを契約した場合、CONSULT料金をBOOSTから全額差し引き

### ファネル設計

```
Visitor（無料）
    ↓ 週次ライブ体験
BASIC参加（月2,980円）
    ↓ もっと深掘りしたい
ACTIVE参加（月9,800円）
    ↓ 事例に興味・合意形成が必要
PRIME入会（年200万円）+ 20%割引獲得
    ↓ 具体的に相談したい
CONSULT（クイック/スタンダード）
    ↓ 本格導入したい
RECIPES活用（手順書） または BOOST実施（プロジェクト）
    ↓ 成果が出た
PRIMEで事例共有・次の勝ち筋へ
    ↓
ケース公開・横展開
```

### 価値約束

→ 詳細: `baao/training/project.md` § 00_competitive

- **収益**: 研修売上、Unsonハンドオフ数

## 00_reality_check｜事業の現実認識

### この事業の本質的な問い

1. **誰のPLなのか？**
   - BAAO法人として収益を上げるのか
   - 山本個人の講師業として稼ぐのか
   - Unsonへのリード獲得チャネルとして機能させるのか
   → **現状未決定。初回セミナー後に決める。**

2. **キャリアパートナーズ依存の構造**
   - リード獲得: キャリアパートナーズ
   - 集客・告知: キャリアパートナーズ
   - 運営事務: キャリアパートナーズ
   - 請求・入金: キャリアパートナーズ
   → **BAAOは「講師派遣 + コンテンツ供給」に近い立場**

3. **「研修屋」から抜けられるか？**
   - 現状の見え方: 「AIガバナンスに強い、よくある生成AI研修」
   - 差別化の根拠: 山本の権威性（経産省ガイドライン執筆協力）
   - 課題: 権威性が「山本個人」に紐づいており、BAAOのプロダクトになっていない

### 致命的に足りていないもの

1. **ユニットエコノミクス**: 1案件でBAAOにいくら残るか不明
2. **顧客所有権**: キャリアパートナーズ経由の顧客はBAAO/Unsonでフォローできるのか
3. **講師育成計画**: 山本依存からの脱却プランなし
4. **LTV設計の数値根拠**: ファネル各段階のコンバージョン率が楽観的

## 02_business_model｜ビジネスモデルの選択肢

### Option A: 講師派遣モデル（現状の延長）

```
キャリアパートナーズが顧客開拓・運営
    ↓
BAAOは講師（山本）を派遣
    ↓
売上の40〜60%がBAAO取り分
```

**メリット**:
- 営業・運営負荷がほぼゼロ
- キャリアパートナーズの顧客基盤を活用

**デメリット**:
- 顧客リストはキャリアパートナーズ所有
- Dojo/Unson送客の自由度が低い
- 価格決定権がない
- スケールしにくい（山本の稼働上限）

**向いているケース**:
- 山本の講師業としての副収入
- 事業としてではなく個人の活動

### Option B: 自社事業モデル（独立路線）

```
BAAO自身がリード獲得・運営
    ↓
売上100%がBAAO
    ↓
顧客リストをBAAO/Unsonで活用
```

**メリット**:
- 顧客所有権がBAAO
- Dojo/Unson送客を自由に設計
- 価格決定権がBAAO

**デメリット**:
- 営業・運営の負荷が大きい
- リード獲得チャネルをゼロから構築
- 初期投資が必要

**向いているケース**:
- Unsonへのリード獲得チャネルとして本気でやる場合

### Option C: ハイブリッドモデル（推奨）

```
Phase 1: キャリアパートナーズ経由で実績構築
    ↓
Phase 2: 実績を武器にBAAO独自チャネルを開拓
    ↓
Phase 3: キャリアパートナーズ依存度を下げつつ両立
```

**条件**:
- キャリアパートナーズ経由の顧客にもDojo案内が可能（合意必須）
- 研修参加者リストの共有範囲を明確化

## 03_partner_negotiation｜キャリアパートナーズとの交渉ポイント

### 合意すべき事項

| 項目 | BAAO希望 | 交渉優先度 |
|------|---------|-----------|
| **収益分配率** | BAAO 60%以上 | 高 |
| **顧客リスト共有** | 研修参加者リストをBAAOにも提供 | 最高 |
| **Dojo案内の許可** | セミナー終了時にDojo案内可 | 最高 |
| **追加営業の許可** | 研修後のBAAO/Unson営業可 | 高 |
| **価格決定権** | 有料研修の価格はBAAOが決定 | 中 |
| **独自開催の許可** | BAAO単独でのセミナー開催可 | 中 |

### 交渉の優先順位

1. **顧客リストとフォロー権限**（これがないとUnson送客の意味がない）
2. **収益分配率**（60%以上確保）
3. **Dojo案内の許可**

### 交渉が決裂した場合

- キャリアパートナーズ経由は「山本の講師業」として割り切る
- BAAO事業としては独自チャネル構築に集中

## 01_strategy｜戦略骨子

### プロダクト概要（1文）
パーソナライズセールスレターを高速大量生成し、自動PDCA機能とCS伴走で商談数を最大化させるAI基盤。

### ICP（1文）
新規開拓に課題を持つBtoB企業（年商3億円以上×顧客単価100万円以上）で、営業の「量」と「質」を両立したいチーム。

### 約束する価値（1文）
もう"できる人"に頼らなくていい。顧客調査93%削減、レター作成15秒、商談化率13倍以上で、量と質どちらも上げて商談数を最大化。

### プロダクト3つの特徴

#### 1. パーソナライズレター生成
ターゲット企業毎に最適なセールスレターを高速大量生成。
- Webサイトをクローリングし、企業情報を自動収集
- LLMで各社に最適化されたレターを生成（平均15秒/通）
- トップセールスが15-30分かけて作成する品質をAIが再現

#### 2. 自動PDCA機能
KPI可視化・結果分析・改善提案で、プロジェクト毎の次の手を提案。
- 計画設定 → 高速生成/送信 → 詳細分析 → 改善提案のサイクル
- 数値データに基づく効果測定（メッセージ効果、送信タイミング効果等）
- 質的データに基づく洞察（ユーザーフィードバック分析）

#### 3. 営業アドバイザー（CS）伴走
営業のプロフェッショナルが新規商談獲得のためのアプローチ最適化をサポート。
- **アプローチターゲットと送付件数の策定**: 既存顧客リストとアウトバウンドのバランスを相談の上で計画
- **アプローチに最適な文面生成支援**: ターゲットに対して最適な文面を選定
- **商談獲得まで見据えたオペレーション**: 顧客フォローやISに関するアドバイス、または運用代行
- **アプローチ結果に基づいた改善提案**: 件名・訴求コンテンツ・言い回しの変更を実施

### 競合優位性・強み
- **代表堀氏のレターノウハウ**: 元スーパー営業マン。パーソナライズレターで返信率10%を達成した実績。このルールをデータ化・チェックリスト形式で品質担保
- **66万件のリスト保有**: Web情報で随時更新し精度向上中
- **プロダクト思想**: 人間らしさ・エモさを残しつつAI活用。営業の付加価値が高い部分は人間が担う設計
- **国内専門プレイヤー不在**: オールパーソナライズ×質的分析/改善を専門とする競合は日本国内に不在

## 04_delivery｜デリバリー・運用

### TTFV ゴール
- Day0: DNS/SPF/DKIM/DMARC設定完了

- Day30: 初回レポート＋AB改善案提出（開封/返信/アポ率の改善確認）

### 標準ステップ
1. ドメイン認証・送信元設定（バウンス/スパム対策含む）
2. リスト/セグメント取り込み、NGワード・法務チェック
3. テンプレ初期投入＋テスト送信
4. 本番送信開始（量は段階的にウォームアップ）
5. 週次レポートとABテスト、30日目に改善サイクルを回す

### 送信チャネル別技術構成（2025-11 MTG更新）

#### メール送信
- **技術**: SendGrid API
- **状況**: 到達率良好、負荷は低い
- **送信量**: 顧客あたり月1,000通〜（ミニマム）、多い顧客は数千通

#### フォーム送信
- **技術**: Playwright（サーバー内で動作）
- **課題**:
  - サーバー負荷が高い
  - 到達率向上のための施策が必要
  - フォームの場所特定にLLMを使用
- **対策検討**: サーバーリソース最適化、到達率改善施策

### レター生成パイプライン

1. **リスト取得**: 66万件のリストから対象抽出
2. **Web情報収集**: 顧客のWebページからLLMで情報抽出
   - 会社概要、サービス内容、ニュースなど
   - フォームの場所特定
3. **レター生成**: Claude 3.5 Sonnetでパーソナライズレター作成

4. **品質チェック**: （構築中）LLMジャッジによる品質評価
5. **送信**: メール or フォーム送信

### 運用監視・メンテナンス体制（課題）

#### KPI定義
- **GRR (Gross Revenue Retention)**: 既存顧客からの収益維持率（チャーン/コントラクション除く）
- **NRR (Net Revenue Retention)**: 既存顧客からの純収益成長率（エクスパンション込み）
- **LTV**: 顧客生涯価値（660万円固定）
- **CAC**: 顧客獲得コスト（マーケ費+採用費+人件費按分）
- **Rule of 40**: 売上成長率 + EBITDA Margin（40%以上が健全）

3. パートナー送客：ハピホテ/カップルスの施設ページからAitle投稿への導線を交渉（相互送客）。
4. 口コミ投稿誘因：アプリ内で宿泊後のリマインド通知→投稿/レビュー→タイムライン露出。

### 主要フック（コピー骨子）
- キャッチ：URLとロゴだけ。今日行けるホテルが3分で見つかる＆相談できる。
- リード：地図・条件・マッチングの3検索＋タイムラインで実体験が見える。Shadow Callで不安も即解消。
- ボディ：写真投稿・保存リスト・共有リンク・AI音声相談のワンストップ。無料で使えて、広告なし/優先相談はプレミアム。

### グロースループ
1. 検索/閲覧 → 保存/共有 → 来訪 → 投稿 → タイムライン露出 → 新規獲得（UGCループ）
2. Shadow Call体験 → 安心感 → 予約リンククリック → 成果報酬 → クーポン配布 → リピート

### オンボード（3分以内のハッピー）
1. Googleログイン
2. 好みを2タップ入力（エリア/目的/予算）
3. マッチング結果から1件保存 or 共有
4. Shadow Call を1回試す（30秒でも可）

### 反論ハンドリング（想定）
- 「広告が多い？」→ プレミアムで非表示。無料でも使用感を損なわない頻度に制御。
- 「信頼できる情報？」→ 投稿は写真必須＋通報/非表示機能。公式情報は元URL明記。
- 「予約はできるの？」→ 予約は提携先リンクで実行（今後アプリ内決済を検討）。

- 初回起動〜初回検索→保存/共有まで「3分以内」。
- 初回Shadow Callを当日中に体験させる。

### オンボードフロー
1. ログイン（Google） → 好み/目的を2タップ。
2. マッチング検索結果を表示、1件保存 or 共有。
3. Shadow Call をワンタップ試用（30秒サンプル）。
4. 通知/メールで「投稿リマインド」と「保存先クーポン」配布。

### コンテンツ/UGC運用
- 投稿は写真必須＋通報/非表示機能。スパム/不適切判定をAI＋HITLで運用。
- タイムラインは最新＋人気の二軸。直近投稿を優先露出して回遊を促す。

### サービスレベル/品質管理
- LCP/CLS監視と広告頻度の上限（UXを壊さない閾値を設定）。
- Shadow Call SLA（応答遅延/エラー率）を日次で計測。
- 不適切投稿・権利侵害の対応SLA（24h以内一次対応）。

- UGCの信頼性 → 通報・ブロック・投稿者BANの明文化、AIフィルタ＋人の確認。
- 広告によるUX悪化 → 表示密度とレイテンシを週次レビュー、LCP悪化時は即停止。

ハピホテ/カップルス/自社URLとロゴを渡すだけで、当日にリッチな公式HPが自動生成され、最新情報がAI/HP/OTAへ同時反映。夜勤店長の手間ゼロで直販とAI露出を一気に伸ばす。

### ICP（1文）
公式HPがない/更新停滞中のラブホテルオーナー・支配人。OTA手数料と電話過多に課題を持ち、直販を増やしたいが制作・運用リソースが乏しい層。

### 約束する価値（1文）
5分で無料公式サイトを開設し、入力データをAI検索・独自ドメインHP・AIチャット・決済に自動反映、PoC 3–5か月で直販+5pt/人件費▲4〜5万円を狙う。

### ブランドキーワード（3本柱）※わかりやすく言い切る
