# 01_strategy｜戦略骨子

## カタリスト育成プログラム 戦略（骨子）

- 北極星: 「広義のPoCとフィットジャーニーを標準化し、中間層カタリストが自走で価値検証できる状態」を作る。
- 対象とゴール
  - 主対象: 中間層カタリスト60%（初学〜実務中級）。副対象: 専門性カタリスト／上位層はショーケース役。
  - ゴール: ①広義PoC/フィットジャーニーを案件で適用できる ②BBX/営業と連携しフォローPoCを立ち上げられる ③専門性カタリストをハブに案件を加速。
- 期間ロードマップ（案）
  - 12月: 改訂WS（1h45–2h）実施、広義PoC教材パックv1配布、週次ニュース満足度アンケート開始
  - Q1: 広義PoC導入率30%（参加者の案件で適用）、勉強会録画をオンボーディング教材として定常配信
  - Q2: フォローPoC化率10%（勉強会参加案件の10%がPoC提案へ）、専門性カタリスト活用件数を月次でトラッキング
- スキルマップ（簡易）
  - 知識: 生成AIトレンド、業界ニュース、BBXケース
  - プロセス: フィットジャーニー、広義PoC(POV)設計、価値仮説→検証計画
  - ツール: DialogAI/CONNの基本操作、ニュース配信パイプライン
- リスク/手当て
  - 参加率低下 → 事前デモとペアレビューで「やらされ感」低減、再参加率をKPI化
  - 習熟度差 → 初学者用オンボーディング（録画＋QA）、専門性カタリストによる添削枠
  - 成果不明瞭 → フォローPoC化率・導入率を必ず計測し定例で共有

### 2025-11-26 NTT R&D FORUM 視察サマリー（Docomoビジネス向け）

- tsuzumi v2 は「日本企業向けミッドサイズ＋オンプレ特化」へ舵切り。金融など規制ドメインでGPU1枚運用が前提で、汎用LLM競争から降りる姿勢が鮮明。
- 音声to音声モデル＋記憶アーキテクチャ（エピソード/長期記憶＋ユーザー情報DB）が国内トップクラス。コンタクトセンターDXでは閉域・低レイテンシの強みが大きい。
- マルチエージェント・会議支援は技術的には横並びだが、閉域運用と社内データ連携を軸に「会議OS」へ物語を再構成する余地あり。
- エッジAIカメラボード（20–25万円級、メタデータ送信のみ）などセンシング＋ネットワークのアセットが強力。LLM単体より「現実世界のデータを握る」戦略が核心。
- IOWN/光技術・デジタルトラスト（HSM、分散ID、ユニバーサルウォレット）・宇宙/HAPSは中長期だが、AIエージェントの法的/セキュア実行基盤としてストーリーを組める。

#### 推奨アクション（このプロジェクトで扱う観点）
1. tsuzumiを「閉域×オンプレ×規制産業向けプラットフォームコンポーネント」と定義し、研修・教材ではGPT比較より“現場で回るLLM”を強調する。
2. 週次ニュース/研修に「音声対話スタック特集」を追加し、コンタクトセンターDXの標準構成（音声to音声モデル＋記憶アーキテクチャ）をパッケージ化。
3. 勉強会演習で「会議OS」シナリオを導入し、マルチエージェントを“意思決定ログ資産化”文脈で説明する。
4. エッジAIデバイス×LLMのユースケースを教材に追加し、「センシング→メタデータ→LLM」の標準アーキテクチャを図解（IoT/防災/工場安全など）。
5. デジタルトラスト/HSM/IDを基礎知識として押さえ、AIエージェントが契約・決裁に関与する前提でのセキュリティ講座を設ける。

# 01_strategy｜戦略骨子

## 4周年イベント（OPEN HUB 4th）戦略

- 目的: 当初は2025年2月OPEN HUB 4周年イベントでCONN/セールステックAIエージェントを披露する計画だったが、**2025-11-25定例で「4周年イベントは実施しない（スケジュール自由化）」が共有**。イベント自体は消滅したが、ショーケース価値は維持するため、**「代替ショーケース（社内/顧客向け）」としてデモを完成・披露しPoC導線を作る**。
- デモコンセプト（議事録から確定している軸）
  - CONNちゃん（デジタルヒューマン）＋セールステック/オープンハブエージェント。
  - 機能: アジェンダ自動生成 → 論点整理 → 議論中サジェスト → クロージングでネクストアクション・役割・期限を提示（10/27四周年向け会議、10/27提案書案）。
  - マルチロケーション・遠隔協働デモ（複数拠点/芸人・オーケストラ例などで「距離を超えた共創」を訴求, 10/24議事）。
- 現状
  - シナリオ未確定のまま開発が止まった経緯（10/28 Exaから「シナリオが無いと要件が決まらない」と指摘）。
  - 11/25時点で期日拘束なし。**こちらから新披露日とミニマムスコープを提示する必要あり**。
  - セキュリティ前提（12/19・3月設定変更提出）は継続。GitHub利用可否は運用チーム確認が必要（00_context参照）。
- マイルストーン再設計（提案）
  - 12月第1週: シナリオ確定（5分版/15分版）、役割分担A/B/C決定、デモ範囲凍結。
  - 12月末: 社内クロス拠点デモ（録画取得）、フィードバック反映。
  - Q1: 顧客向けショーケース（OPEN HUB館内/Virtual、もしくはDocomo Business向け小規模イベント）で披露。
- リスクと手当
  - モメンタム低下 → 披露日を再設定し週次で進捗可視化。
  - シナリオ不在 → 12月第1週で決着、Exa/UIチームと合同レビュー。
  - LLM品質・記者対応リスク → シナリオ/ドメイン限定＋安全プロンプト、オフライン/閉域環境で実施。
  - UI/ビジュアライザー依存 → 早期に必要素材・レンダリング担当を決定。

# 03_sales_ops｜営業オペ

## ショーケース 営業オペ

- 目的: 4周年イベント中止後も、デモを軸に案件化。デモ→質疑→ミニPoC提案の3ステップを小回りで回す。
- ステップ
  1. 招待/アジェンダ共有（5分版/15分版いずれかを明示）。招待文に「OPEN HUB 4th計画の代替ショーケース」である旨を記載。
  2. デモ実演
     - オープニング: 課題と体験ゴール提示
     - 本編: CONNによる論点整理→議論サジェスト→要約/次アクション提示（シナリオ固定で安全運用）
     - クロージング: アクションプラン提示＋PoCオプション案内
  3. フィードバック収集→次アクション設定（PoC案、評価指標、日程）
- 必携アセット
  - デモ台本（5分/15分）
  - ビジュアルセット（マルチロケーション表示、アクションボード）
  - 技術概要1pager＋構成図（Exa連携明記）
  - セキュリティFAQ（12/19提出物と整合）
  - PoC提案テンプレ（稟議1枚シート）
- 計測
  - デモ実施数 / 参加者数 / 再演依頼数
  - PoC提案数 / PoC受注率 / PoC→本導入転換率
