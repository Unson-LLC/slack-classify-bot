# 01_strategy｜戦略骨子

## Aitle 戦略メモ（toC版骨子）

### プロダクト概要（1文）
ホテル検索・思い出共有・AI音声相談をワンアプリで提供し、URLと写真だけで「今すぐ行ける・共有できる」体験を作る。

### ICP（1文）
18〜35歳のカップル・女子旅・推し活ユーザーで、深夜/直前でも安心して泊まれるホテルを探し、友達と共有したい人。

### 約束する価値（1文）
3分で好みのホテルを見つけて保存・共有し、その場でAI音声相談（Shadow Call）もできる“今すぐ使える”宿選び体験。

### 収益モデル（現時点の仮説）
- 無料利用をベースに、広告/アフィリエイト送客（予約リンク）で収益化。
- 将来のプレミアム案：広告非表示、優先AIコンシェルジュ、限定クーポン配布。

### TTFV目標
- 初回起動〜初回検索・1件保存/共有までを「3分以内」。
- Shadow Call 初回利用を「初回体験当日」に誘導。

### 主要KPI案
- WAU/MAU、D1/D7リテンション
- 検索→詳細閲覧→保存/共有CVR
- 投稿数・投稿閲覧数・いいね/コメント率
- Shadow Call 利用率・満足度
- 送客クリック数・予約コンバージョン（アフィリエイト用）

### 次のアクション
- 02_offerで無料/広告モデルと将来プレミアムの線引きを整理
- 03_sales_opsを「グロースオペ」に転換（獲得チャネルとループ設計）
- 04_deliveryにオンボ体験（3分以内のハッピー）とShadow Call誘導を記述
- 05_kpiをBtoC指標に差し替え（リテンション・CVR・コンテンツ投稿）

# 02_offer｜価格・オファー

## 02_offer (Aitle toC) — 価格・オファー草案

### 提供形態
- 無料プラン（デフォルト）：検索・保存・共有・投稿・Shadow Callの基本利用。広告表示あり。
- プレミアム（案）：月額/年額で広告非表示＋Shadow Call優先枠＋限定クーポン（実装検討中）。

### マネタイズ
- 広告：タイムライン/検索結果のネイティブ広告、軽量バナー（頻度上限を設定）。
- 送客/アフィリエイト：予約リンククリック経由の成果報酬。
- 将来オプション：ブランドタイアップ（特集面・クーポン配布）。

### 価値メッセージ
- 3分で「行ける」ホテルが見つかり、友達と共有・投稿まで一気通貫。AI音声相談で深夜でも不安を解消。
- 無料のままでも十分使え、広告なしや優先サポートを望むヘビーユーザーはプレミアムでアップセル。

### オファーの方向性
- 無料で当日価値体験（検索→保存/共有→Shadow Call）を提供し、送客と広告で収益化。
- プレミアムは「時間と安心の購入」：広告なし、混雑時の優先相談、限定クーポン。

### TODO / 検証事項
- 広告表示頻度とLCP/UXへの影響を計測し、上限を決定。
- 送客リンクのCVRと主要パートナー（ハピホテ/カップルス/自社HPなど）の実績取得。
- プレミアム価格のWTP調査（広告非表示＋優先Shadow Callで月いくらまで許容か）。

# 03_sales_ops｜営業オペ

## Aitle グロースオペ（toC）

### 獲得チャネル（初期優先度）
1. 自然流入：SNS（X/Instagram/TikTok）で「当日行けるホテルまとめ」「写真映え投稿」を発信。
2. 検索流入：ロングテール「駅名＋ホテル＋深夜」「女子会 ホテル 写真映え」記事／LP。
3. パートナー送客：ハピホテ/カップルスの施設ページからAitle投稿への導線を交渉（相互送客）。
4. 口コミ投稿誘因：アプリ内で宿泊後のリマインド通知→投稿/レビュー→タイムライン露出。

### 主要フック（コピー骨子）
- キャッチ：URLとロゴだけ。今日行けるホテルが3分で見つかる＆相談できる。
- リード：地図・条件・マッチングの3検索＋タイムラインで実体験が見える。Shadow Callで不安も即解消。
- ボディ：写真投稿・保存リスト・共有リンク・AI音声相談のワンストップ。無料で使えて、広告なし/優先相談はプレミアム。

### グロースループ
1. 検索/閲覧 → 保存/共有 → 来訪 → 投稿 → タイムライン露出 → 新規獲得（UGCループ）
2. Shadow Call体験 → 安心感 → 予約リンククリック → 成果報酬 → クーポン配布 → リピート

### オンボード（3分以内のハッピー）
1. Googleログイン
2. 好みを2タップ入力（エリア/目的/予算）
3. マッチング結果から1件保存 or 共有
4. Shadow Call を1回試す（30秒でも可）

### 反論ハンドリング（想定）
- 「広告が多い？」→ プレミアムで非表示。無料でも使用感を損なわない頻度に制御。
- 「信頼できる情報？」→ 投稿は写真必須＋通報/非表示機能。公式情報は元URL明記。
- 「予約はできるの？」→ 予約は提携先リンクで実行（今後アプリ内決済を検討）。

### KPIとの紐付け
- 獲得：新規インストール/WAU、初回検索率、初回保存/共有率
- エンゲージ：投稿率、Shadow Call利用率、タイムライン閲覧→クリック率
- 収益：送客クリック数、アフィリエイトCVR、広告CTR/LCP影響

# 04_delivery｜デリバリー

## Aitle デリバリー（toCプロダクト運用）

### TTFV ゴール
- 初回起動〜初回検索→保存/共有まで「3分以内」。
- 初回Shadow Callを当日中に体験させる。

### オンボードフロー
1. ログイン（Google） → 好み/目的を2タップ。
2. マッチング検索結果を表示、1件保存 or 共有。
3. Shadow Call をワンタップ試用（30秒サンプル）。
4. 通知/メールで「投稿リマインド」と「保存先クーポン」配布。

### コンテンツ/UGC運用
- 投稿は写真必須＋通報/非表示機能。スパム/不適切判定をAI＋HITLで運用。
- タイムラインは最新＋人気の二軸。直近投稿を優先露出して回遊を促す。

### サービスレベル/品質管理
- LCP/CLS監視と広告頻度の上限（UXを壊さない閾値を設定）。
- Shadow Call SLA（応答遅延/エラー率）を日次で計測。
- 不適切投稿・権利侵害の対応SLA（24h以内一次対応）。

### リスクと対策
- 予約は外部リンク依存 → リンク死/在庫ズレの監視アラートを入れる。
- UGCの信頼性 → 通報・ブロック・投稿者BANの明文化、AIフィルタ＋人の確認。
- 広告によるUX悪化 → 表示密度とレイテンシを週次レビュー、LCP悪化時は即停止。

### TODO
- Shadow Call のFAQテンプレ/想定質問を整備し、満足度計測を実装。
- 投稿リワードのABテスト（バッジ/クーポン/優先表示）。
- インバウンド向け多言語UIの段階導入計画を作成。
