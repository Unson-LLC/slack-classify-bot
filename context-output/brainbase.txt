# 01_strategy｜戦略骨子

## brainbase 01_strategy｜共通リポジトリOS

### プロダクト概要（1文）
全事業を統一OS（判断基準・テンプレ・KPI・RACI・ブランド資産）で動かし、創業者が現場にいなくても週次30分レビューだけで継続運転できる状態をつくる。

### ICP（1文）
UNSON / BAAO / SalesTailor / TechKnight / Zeims / Aitle など、複数事業を同時に扱い、**引き継ぎ・再現性・立ち上げ速度**が求められる GM / PO / PM / 社内オペレーション担当。

### 約束する価値（1文）
ナレッジ・判断基準・タスク・ブランド資産を brainbase に一本化し、01–05戦略・RACI・テンプレ・KPIを標準化することで、**「90日で新規事業を仕組み化」「人が抜けても運営が止まらない」**状態を保証する。

### 価格・前受け（内部運用方針）
前受け 100%（例外なし）。  
情報はすべて `_codex` に集約し、タスクは `_tasks/index.md` を唯一の正とする。

### TTFV目標
新規事業は 90日以内に **01–05（戦略）・RACI・ブランドガイド・KPI・会議リズム（週次/四半期）・前受けルール** をフル実装し、自律運転できる状態をつくる。

### 主要KPI案
- タスク一本化率（_tasks への集約率）
- 01–05 充足率（事業ごとの OS 完成度）
- RACI / AGENTS 運用率（役割定義と行動ルールの適用度）
- ブランドガイド整備率（org 単位）
- 引き継ぎリードタイム（新規メンバーが理解→運用できるまでの日数）

### 次のアクション
- 02_offer / 03_sales_ops / 04_delivery / 05_kpi を「内部OS視点」でドラフト化  
- RACI / AGENTS の適用状況をレビュー（BRAINBASE-W3）

# brainbase｜MVV（共通リポジトリのOS）

## Mission（創業の理由｜今〜3年）
創業者が現場から抜けても、全事業が同じOSで動き、週次30分のレビューだけで運営が回る状態をつくる。判断基準・ナレッジ・タスク・ブランド資産を brainbase に一元集約し、誰が見ても同じ判断ができるようにする。

## Vision（未来像｜3〜5年）
brainbase 上に、すべての事業（UNSON / BAAO / SalesTailor / TechKnight / Zeims / Aitle）が、共通の構造・KPI・役割分担（RACI）・ブランド資産で接続される。  
その結果として、**「人が抜けても回る」「新規事業が90日で立ち上がる」**状態を標準化する。

## Values（現在の行動指針）
- 一元管理：情報は brainbase に集約し、リンクで参照する。二重管理はしない。
- 30分レビュー基準：週次レビューで30分あれば追える粒度で書く。迷ったら削る。
- テンプレ優先：テンプレ化・再利用可能な形を最優先する。例外対応は「追加コスト」として扱う。
- 数字で回す：KPIとゲート（次のステージに進む条件）を定義し、感覚ではなくデータで判断する。
- 前受けとルール徹底：前受け・SLA・割引禁止ルールを明文化し、例外は必ず決裁ログを残す。

## 補足
- ブランドガイド：`_codex/brand/`（外向き表現）、ロゴ正本は `_codex/brand/assets/` に保存する。
- プロジェクト戦略：`_codex/projects/` に 01–05 のフレームで整理して格納する。
- タスク：`_tasks/index.md` を唯一のタスク一覧とし、他ではタスクを持たない。
